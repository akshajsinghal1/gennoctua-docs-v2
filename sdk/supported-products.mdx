## Photo Requirements by Product

### Face Photo Required

These products overlay onto the face or neck area.

| Product | `productType` |
|---|---|
| Sunglasses | `sunglasses` |
| Eyeglasses | `eyeglasses` |
| Necklaces / jewellery | `jewellery` |
| Earrings | `earrings` |
| Lipstick | `makeup_lipstick` |
| Foundation | `makeup_foundation` |
| Mascara | `makeup_mascara` |

<Info>Accepted categories: `male_face_closeup`, `female_face_closeup`, `child_face_closeup`</Info>

---

### Full Body Photo Required

These products are placed on the full body.

| Product | `productType` | Gender restriction |
|---|---|---|
| Men's clothing | `mens_clothing` | Male only |
| Women's clothing | `womens_clothing` | Female only |
| Kids' clothing | `kids_clothing` | Child only |
| Shoes / sneakers | `footwear` | Any gender |
| Handbags / purses | `bags` | Any gender |

---

### Room Photo Required

These products are placed into a room scene. The user uploads a photo of their room instead of a person photo.

| Product | `productType` | Room type detected |
|---|---|---|
| Bedroom furniture | `bedroom_furniture` | Bedroom |
| Living room furniture | `living_room_furniture` | Living room |
| Dining room furniture | `dining_room_furniture` | Dining room |

<Info>Accepted categories: `room_bedroom`, `room_living_room`, `room_dining_room`</Info>

The SDK automatically detects the room type from the uploaded photo using on-device AI (YOLOv8 object detection). The user does not need to label their photo.

## Checking Eligibility

```ts
const eligibility = sdk.checkEligibility("sunglasses");

if (eligibility.eligible) {
  // Show Try On button
} else {
  switch (eligibility.reason) {
    case "REQUIRED_USER_IMAGE_MISSING":
      // User hasn't uploaded the right photo yet
      break;
    case "PRODUCT_TYPE_UNSUPPORTED":
      // Phase 2 product — not yet available
      break;
    case "PRODUCT_TYPE_NOT_FOUND":
      // productType was not provided
      break;
  }
}
```

## Best Practices for Product Images

<CardGroup cols={2}>
  <Card title="Use white background" icon="image">

  </Card>

  </Card>
  <Card title="Front-facing view" icon="eye">

## Internal Mapping

When you call `sdk.personalize({ imageUrl, productType })`, the SDK maps your `productType` to a category and label sent to HP.

| `productType` | Category sent | Label sent |
|---|---|---|
| `sunglasses` | `eyewear` | `sunglasses` |
| `eyeglasses` | `eyewear` | `eyeglasses` |
| `mens_clothing` | `clothing` | `mens clothing` |
| `womens_clothing` | `clothing` | `womens clothing` |
| `kids_clothing` | `clothing` | `kids clothing` |
| `footwear` | `footwear` | `footwear` |
| `jewellery` | `jewellery` | `jewellery necklace` |
| `earrings` | `jewellery` | `earrings` |
| `bags` | `accessories` | `bag` |
| `makeup_lipstick` | `makeup` | `lipstick` |
| `makeup_foundation` | `makeup` | `foundation` |
| `makeup_mascara` | `makeup` | `mascara` |
| `bedroom_furniture` | room generation | `bedroom` |
| `living_room_furniture` | room generation | `living_room` |
| `dining_room_furniture` | room generation | `dining_room` |

You don't need to know any of this — just pass the `productType` value and the SDK handles the rest.
