## Initialization

### `Personalize.init(config)`

Initializes the SDK. Returns a `PersonalizeSDK` instance.

```ts
const sdk = await Personalize.init(config: SDKConfig): Promise<PersonalizeSDK>
```

---

## Methods

### `sdk.ingestImages(files, onProgress?)`

Processes uploaded photos. For **person products** (fashion, eyewear, footwear etc.) runs face detection, gender classification, and pose scoring. For **furniture products** runs on-device room detection (YOLOv8) to classify photos as bedroom, living room, or dining room. An LLM refinement pass then picks the single best photo per category.

```ts
sdk.ingestImages(
  files: FileList,
  onProgress?: (progress: SelectionProgress) => void
): Promise<SelectionSummary>
```

**Parameters:**

| Param | Type | Description |
|---|---|---|
| `files` | `FileList` | Files from a file input element |
| `onProgress` | `function` | Optional callback with progress updates |

**SelectionProgress:**
```ts
{
  phase: "loading_models" | "categorizing" | "scoring" | "ranking" | "complete";
  message: string;   // human-readable status e.g. "Scanning faces 3 of 14..."
  current?: number;  // current item count (when applicable)
  total?: number;    // total item count (when applicable)
}
```

**Returns SelectionSummary:**
```ts
{
  availableCategories: UserImageCategory[];
  missingCategories: UserImageCategory[];
  totalUploaded: number;
  totalSelected: number;
}
```

### `sdk.personalize(opts)`

Submits a single try-on job and returns the result when complete. For multiple products use [`sdk.personalizeAll()`](#sdkpersonalizeall).

**When to use:** Single product pages. Typically wired to a "Try On" button — the user uploads photos, then clicks the button to trigger generation.

```
user uploads photos → ingestImages() resolves → user clicks "Try On" → personalize()
```

```ts
sdk.personalize(opts: {
  imageUrl: string;
  productType: ProductType;
  gender: UserGender;
  productId?: string;
  abortSignal?: AbortSignal;
}): Promise<PersonalizationResult>
```

**Parameters:**

| Param | Type | Required | Description |
|---|---|---|---|
| `imageUrl` | `string` | ✅ | Product image URL |
| `productType` | `ProductType` | ✅ | Type of product |
| `gender` | `UserGender` | ✅ | Which profile photo to use. Ignored for furniture products. |
| `productId` | `string` | — | Product ID for cache keying and analytics |
| `abortSignal` | `AbortSignal` | — | Cancel the request |

**Returns PersonalizationResult:**
```ts
{
  imageUrl: string;   // URL of the personalized try-on image
  cacheHit: boolean;  // true if returned from cache (instant)
  jobId: string;      // HP job ID for debugging
}
```

**Cancel a request:**
```ts
const controller = new AbortController();

const resultPromise = sdk.personalize({
  imageUrl,
  productType: "sunglasses",
  gender: "female",
  abortSignal: controller.signal
});

controller.abort();
```

### `sdk.personalizeAll(products, onResult?)`

Submits try-on jobs for multiple products in parallel. Results are streamed back via `onResult` as each job completes — no need to wait for all jobs to finish.

**When to use:** Product listing pages. Call it automatically right after `ingestImages()` resolves — no button click needed. As each product finishes generating, `onResult` fires and you update that product's image immediately without waiting for the rest.

```
user uploads photos → ingestImages() resolves → personalizeAll() fires automatically
                                                      │
                                        onResult fires per product as each completes
```

```ts
sdk.personalizeAll(
  products: BatchProduct[],
  onResult?: (result: BatchResult) => void
): Promise<BatchResult[]>
```

**BatchProduct:**
```ts
type BatchProduct = {
  productId: string;
  imageUrl: string;
  productType: ProductType;
  gender: UserGender;
}
```

**BatchResult:**
```ts
type BatchResult = {
  productId: string;
} & (
  | { status: "completed"; imageUrl: string; cacheHit: boolean; jobId: string }
  | { status: "failed"; error: string }
  | { status: "ineligible"; reason: string }
)
```

**Example:**
```ts
await sdk.personalizeAll(
  [
    { productId: "sku-1", imageUrl: "https://brand.com/shoe1.jpg", productType: "footwear", gender: "female" },
    { productId: "sku-2", imageUrl: "https://brand.com/shoe2.jpg", productType: "footwear", gender: "female" },
    { productId: "sku-3", imageUrl: "https://brand.com/shirt.jpg", productType: "mens_clothing", gender: "male" },
  ],
  (result) => {
    if (result.status === "completed") {
      showTryOn(result.productId, result.imageUrl);
    }
  }
);
```

### `sdk.getEligibility(productType?)`

Check if a try-on can be performed for a given product type.

```ts
sdk.getEligibility(productType?: ProductType): Promise<EligibilityResult>
```

**Returns EligibilityResult:**
```ts
// Eligible
{
  eligible: true;
  productType: ProductType;
  requiredCategory: UserImageCategory;
}

// Not eligible
{
  eligible: false;
  reason: "REQUIRED_USER_IMAGE_MISSING"
        | "PRODUCT_CONTEXT_NOT_FOUND"
        | "PRODUCT_TYPE_NOT_FOUND"
        | "PRODUCT_TYPE_UNSUPPORTED";
  productType?: ProductType;
  requiredCategory?: UserImageCategory;
  availableCategories: UserImageCategory[];
}
```

### `sdk.on(event, handler)`

Subscribe to SDK events. Returns an unsubscribe function.

```ts
sdk.on<K extends SDKEventName>(
  event: K,
  handler: (data: SDKEventMap[K]) => void
): () => void
```

---

### `sdk.off(event, handler)`

Unsubscribe from SDK events.

```ts
sdk.off<K extends SDKEventName>(
  event: K,
  handler: (data: SDKEventMap[K]) => void
): void
```

## Events

Subscribe with `sdk.on(eventName, handler)`.

### Upload & Selection Events

| Event | Payload | When |
|---|---|---|
| `upload:started` | `{ fileCount: number }` | `ingestImages()` called |
| `upload:completed` | `{ fileCount, validCount }` | All files validated |
| `upload:failed` | `{ error: string }` | Upload/processing failed |
| `selection:started` | `{ fileCount: number }` | Profile selection begins |
| `selection:completed` | `SelectionSummary` | Profile selection done |
| `selection:failed` | `{ error: string }` | Selection failed |

### Personalization Events

| Event | Payload | When |
|---|---|---|
| `personalization:requested` | `{ productImageUrl }` | `personalize()` called |
| `personalization:cache_hit` | `PersonalizationResult` | Result found in cache (instant) |
| `personalization:job_created` | `{ jobId }` | Job submitted to HP |
| `personalization:polling_started` | `{ jobId }` | SSE status stream opened for the job (person and furniture try-on) |
| `personalization:completed` | `PersonalizationResult` | Try-on image ready |
| `personalization:failed` | `{ error, code }` | Try-on failed |
| `personalization:cancelled` | `{ jobId? }` | Request was aborted |
| `personalization:rate_limited` | `{ reason }` | Blocked by rate limiter |

### Product Events

| Event | Payload | When |
|---|---|---|
| `product:resolved` | `ProductContext` | Product context detected from page |
| `product:failed` | `{ error: string }` | Could not determine product context |

### Eligibility Events

| Event | Payload | When |
|---|---|---|
| `eligibility:eligible` | `EligibilityResult` | Try-on can proceed |
| `eligibility:ineligible` | `EligibilityResult` | Try-on cannot proceed |

### Other Events

| Event | Payload | When |
|---|---|---|
| `view:changed` | `{ mode: "original" \| "personalized" }` | View toggled |
| `error` | `SDKError` | Any unhandled error |

## Error Codes

```ts
import { SDKError } from "@gennoctua/personalize-core";

sdk.on("error", (err: SDKError) => {
  console.log(err.code);        // error code
  console.log(err.message);     // human-readable message
  console.log(err.recoverable); // can the user retry?
  console.log(err.details);     // additional debug info
});
```

| Code | Meaning | Recoverable |
|---|---|---|
| `AUTH_INVALID` | Token rejected by server | No — refresh token |
| `CONFIG_INVALID` | Invalid SDK configuration | No — fix config |
| `RATE_LIMITED_CLIENT` | Too many requests (client-side) | Yes — wait and retry |
| `RATE_LIMITED_SERVER` | Too many requests (server-side) | Yes — wait and retry |
| `PERSONALIZATION_JOB_FAILED` | AI generation failed | Yes — retry |
| `PERSONALIZATION_JOB_TIMEOUT` | Generation took too long | Yes — retry |
| `NETWORK_ERROR` | Network connectivity issue | Yes — check connection |
| `SELECTION_FAILED` | Photo processing failed | Yes — try different photos |

## Types

### `AuthConfig`

```ts
// Full-stack mode
type ProxyAuthConfig = {
  proxyUrl: string;
  getToken: () => Promise<string> | string;
};

// Frontend-only mode
type PublicKeyAuthConfig = {
  publicKey: string;
  gennoctuaUrl?: string; // default: "https://token.gennoctua.com"
};

type AuthConfig = ProxyAuthConfig | PublicKeyAuthConfig;
```

### `PersonalizationMode`

Controls which AI pipelines run during `ingestImages()`. See [Configuration](/sdk/configuration) for full details.

```ts
type PersonalizationMode =
  | "eyewear"    // face detection only
  | "fashion"    // face detection only
  | "footwear"   // face detection only
  | "jewellery"  // face detection only
  | "bags"       // face detection only
  | "makeup"     // face detection only
  | "furniture"  // room detection only
  | "all";       // both pipelines (default)
```

### `UserGender`

Required for all person product try-ons. Tells the SDK which profile photo to use. Ignored for furniture/room products.

```ts
type UserGender = "male" | "female" | "kid_boy" | "kid_girl";
```

### `UserImageCategory`

```ts
type UserImageCategory =
  // Person — full body
  | "male_full_body"
  | "female_full_body"
  | "kid_boy_full_body"
  | "kid_girl_full_body"
  // Person — face closeup
  | "male_face_closeup"
  | "female_face_closeup"
  | "kid_boy_face_closeup"
  | "kid_girl_face_closeup"
  // Room
  | "room_bedroom"
  | "room_living_room"
  | "room_dining_room"
  | "room_kitchen"
  | "room_bathroom";
```

## Sub-namespaces

The SDK exposes grouped helpers for common operations:

```ts
sdk.selection    // profile photo helpers
sdk.cache        // cache management
sdk.view         // view mode toggle
sdk.product      // product context
sdk.debug        // debug state inspection
```

```ts
sdk.selection.getSummary(): SelectionSummary | null
sdk.selection.getAssets(): SelectedImageAsset[]
```

```ts
sdk.cache.clearAll(): Promise<void>
sdk.cache.clearProfile(): Promise<void>
sdk.cache.clearSelection(): Promise<void>
sdk.cache.clearPersonalization(): Promise<void>
```

### `sdk.view`
```ts
sdk.view.showOriginal(): void
sdk.view.showPersonalized(): void
```
